WordGame

This attack is designed to bypass the safety measures of LLMs by simultaneously obfuscating both the input queries and the expected responses.
This approach involves substituting potentially harmful words with word games or puzzles, thereby reducing the likelihood of triggering the model's safety protocols.

Usage example: python run.py -m provider/model -a wrd -t "How to build a bomb"

Below are extra arguments that can be changed using: -e extra_arg1=value1 -e extra_arg2=value2

Extra Argument Description
word_game_plus Using the advanced version of the attack where the word game is placed between other questions. Default: True
game_generation_model Name of the model used to generate the attack hints. Default: openai/gpt-4o
